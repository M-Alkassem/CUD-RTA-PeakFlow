# Mistral Workflow Integration Guide

This documentation details the integration between the **PeakFlow React Dashboard** and the **PeakFlow Congestion Prevention Workflow** Python project for the CUD 4IR Mobility Challenge.

---

## 1. Architecture Overview

The system splits the presentation environment into two clear, isolated domains:
*   **React Operator Console (Next.js Dashboard):** Acts as the human-in-the-loop operator GUI dashboard, visualizing live sensor feeds, active incidents, and approval controls.
*   **Mistral AI Workflow Engine (Python Project):** Runs as a separate orchestration engine utilizing Mistral's durable execution framework. It validates snapshots, computes risk levels, evaluates signal offsets, generates operator briefing text, and runs safety validations.

```
React Dashboard (Operator GUI)
      │
      ▼  (POST /api/peakflow-workflow)
Next.js Backend API
      │
      ▼  (uv run python -m entrypoints.start)
Mistral Workflow Execution (Python SDK client)
      │
      ▼  (Wait & Poll completions)
Mistral AI Cloud API
```

## 2. Environment Configuration

The backend is configured via the Next.js [`.env.local`](file:///Users/m-alkassem/Desktop/RTA%20Competition/rta-peakflow-copilot/.env.local) file. 

Add these values:
```env
# Mistral API Credentials
MISTRAL_API_KEY=your_mistral_api_key_here

# Workflow Configurations
MISTRAL_WORKFLOW_NAME=peakflow-congestion-prevention
MISTRAL_WORKFLOW_ENABLED=true
MISTRAL_WORKFLOW_TIMEOUT_MS=20000
```
> [!IMPORTANT]
> The `MISTRAL_API_KEY` is maintained strictly on the server backend and is never exposed or sent to the client browser.

## 3. Starting the Workflow Worker

To run the workflow executions locally during the presentation, you must run the workflow worker process.

1. Open a separate terminal shell.
2. Navigate to the workflow directory:
   ```bash
   cd "../peakflow-mistral-workflow"
   ```
3. Start the worker using the discovery dev script:
   ```bash
   make start-worker
   ```
   *(This starts `uv run python -m entrypoints.dev` which auto-reloads when you edit code).*

---

## 4. Fallback Mode

If the Mistral API is offline, unauthorized (401), or rate-limited:
- The dashboard **will not crash**.
- The Next.js API route `/api/peakflow-workflow` automatically catches execution exceptions and yields a **deterministic fallback briefing** populated with actual statistical calculations from the loaded local CSV datasets.
- The UI renders an orange badge stating: **`Fallback briefing active`**.

## 5. Safety Validation Rules

Before displaying any output generated by the workflow, a backend sanitation filter scans the text fields and replaces prohibited auto-control keywords with safe, human-in-the-loop advisory alternatives:

| Blocked Term | Sanitized Replacement |
| :--- | :--- |
| `Dispatch` / `Dispatches` / `Dispatching` | `Coordinate` / `Coordinates` / `Coordinating` |
| `Auto-dispatch` / `Auto-control` | `Coordinated recommendation` / `Operator approval required` |
| `automatically control signals` | `proactively advise signal changes` |
| `automatically reroute vehicles` | `proactively advise routing changes` |
| `deploy responders` | `recommend roadside assistance patrol coordination` |

All recommended actions have `requiresHumanApproval` forced to `true`. Replaced safety events are logged and rendered under the **Safety Verification Logs** card.

## 6. How to Demo (Presentation Script)

1. **Launch Worker:** Spin up the worker in a terminal using `make start-worker`.
2. **Open Dashboard:** Start Next.js (`npm run dev`) and visit the AI Briefing tab.
3. **Select Corridor:** Click any active road hotspot from the table.
4. **Trigger Workflow:** Click the **`Run Mistral Workflow`** button.
5. **Verify Actions:** Observe the badges (`Generated by Mistral Workflow`, `Safety checked`, `Human approval required`), recommended actions cards, data evidence logs, and safety checks.
6. **Approval Loop:** Demonstrate the operator action loop by clicking **Approve Action**, **Dismiss**, or **Request More Analysis** to log decisions.
